안녕하세요!
오늘은 자바에서 사용되는 매우 유용한 자료구조인 집합(Set)과 맵(Map)에 대해 알아보겠습니다.

먼저, '집합(Set)'에 대해 이야기해볼까요?
집합(Set)은 중복된 원소를 허용하지 않는 자료구조입니다.

즉, 모든 원소가 유일해야 하죠. 자바에서는 HashSet, TreeSet 등 다양한 Set 구현체를 제공합니다. 가장 많이 사용되는 HashSet을 예로 들어보겠습니다.

집합(Set)의 특징과 사용법


  • 1. 중복 없음 : 집합 내에서는 동일한 값을 가진 원소가 두 번 존재할 수 없습니다.

  • 2. 순서 없음 : HashSet의 경우, 원소들이 특정 순서로 저장되지 않습니다. 


import java.util.HashSet;
import java.util.Set;

public class SetExample {
    public static void main(String[] args) {
        Set<Integer> numberSet = new HashSet<>();
        numberSet.add(1);
        numberSet.add(2);
        numberSet.add(3);
        numberSet.add(2); // 중복 원소는 추가되지 않음

        System.out.println(numberSet); // [1, 2, 3] 순서는 보장되지 않음

        // 원소 포함 여부 확인
        if (numberSet.contains(2)) {
            System.out.println("2는 Set에 있습니다."); // 2는 Set에 있습니다.
        }
    }
}






이제, '맵(Map)'에 대해 알아볼까요?

맵(Map)은 키(key)와 값(value)의 쌍으로 데이터를 저장하는 자료구조입니다.
각 키는 유일해야 하며, 이를 통해 특정 값을 빠르게 찾을 수 있습니다.

자바에서는 HashMap, TreeMap 등 다양한 Map 구현체를 제공합니다. 여기서는 HashMap을 사용해 예시를 들어보겠습니다.

맵(Map)의 특징과 사용법


  • 1. 키와 값의 쌍: 각 데이터는 키와 값의 쌍으로 구성됩니다.

  • 2. 키의 유일성: 키는 중복될 수 없으며, 하나의 키에는 하나의 값만 매핑됩니다. 
      
import java.util.HashMap;
import java.util.Map;

public class MapExample {
    public static void main(String[] args) {
        Map<String, Integer> ageMap = new HashMap<>();
        ageMap.put("Alice", 30);
        ageMap.put("Bob", 25);
        ageMap.put("Charlie", 35);
        ageMap.put("Bob", 26); // Bob의 값을 업데이트

        System.out.println(ageMap); // {Alice=30, Bob=26, Charlie=35}

        // 특정 키에 대한 값 검색
        if (ageMap.containsKey("Alice")) {
          System.out.println("Alice의 나이: " + ageMap.get("Alice"));
        }
    }
}


집합과 맵은 자바에서 데이터를 효율적으로 관리하고 검색하기 위해 자주 사용되는 중요한 자료구조입니다.

집합은 원소의 중복을 방지하고, 맵은 키를 통해 빠르게 데이터에 접근할 수 있는 기능을 제공합니다.

이러한 자료구조를 통해 복잡한 데이터 관리가 필요한 애플리케이션을 보다 쉽고 효율적으로 구현할 수 있습니다.

"문제 : 숫자 카드는 정수 하나가 적혀져 있는 카드이다. 상근이는 숫자 카드 N개를 가지고 있다. 정수 M개가 주어졌을 때, 이 수가 적혀있는 숫자 카드를 상근이가 가지고 있는지 아닌지를 구하는 프로그램을 작성하시오."

이 문제를 해결하기 위해서는, 먼저 상근이가 가진 숫자 카드들을 효율적으로 관리할 수 있는 방법이 필요해요.
 여기서 'HashSet'이 등장합니다.
'HashSet'은 집합을 구현한 자바의 컬렉션으로, 중복을 허용하지 않고 원소의 존재 여부를 빠르게 확인할 수 있습니다.


그럼 HashSet을 이용해 문제를 풀어봅시다!

  1. 1. 숫자 카드 저장: 상근이의 숫자 카드들을 HashSet에 저장합니다. 이렇게 하면, 각 숫자가 존재하는지 아닌지를 빠르게 검사할 수 있어요.

  2. 2. 숫자 확인: 주어진 M개의 숫자에 대해 HashSet을 조회하여 상근이가 해당 숫자 카드를 가지고 있는지 확인합니다. 가지고 있다면 1을, 아니라면 0을 출력합니다.



이 과정을 통해, 우리는 상근이가 특정 숫자 카드를 가지고 있는지 없는지를 효율적으로 확인할 수 있습니다. HashSet을 사용함으로써, 각 숫자의 존재 여부를 빠르게 파악할 수 있죠. 


import java.util.HashSet;
import java.util.Scanner;
import java.util.Set;

public class NumberCard {
    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);

        // 상근이가 가지고 있는 숫자 카드의 개수 N
        int N = scanner.nextInt();
        Set<Integer> cardSet = new HashSet<>();

        // 상근이의 숫자 카드 입력 받기
        for (int i = 0; i < N; i++) {
            cardSet.add(scanner.nextInt());
        }

        // 확인해야 할 숫자의 개수 M
        int M = scanner.nextInt();

        // 확인해야 할 숫자 입력 받고, 상근이가 가지고 있는지 확인합니다
        for (int i = 0; i < M; i++) {
            if (cardSet.contains(scanner.nextInt())) {
                System.out.print("1 ");
            } else {
                System.out.print("0 ");
            }
        }

    }
}



처음에는 어떤 문제에 Set과 Map을 사용해야 할지, 그리고 이들의 메서드가 어떻게 작동하는지 파악하기 어려울 수 있습니다.
특히 집합(Set)과 맵(Map)은 데이터를 저장하고 관리하는 방식이 일반적인 리스트나 배열과는 다르기 때문에, 처음 접하는 사람에게는 다소 생소하게 느껴질 수 있어요.




🤗 여기 몇 가지 팁을 공유할게요 🤗

1. Set을 사용하는 경우 : 중복된 값을 허용하지 않고, 데이터의 유일성을 보장해야 할 때 Set을 사용하세요. 예를 들어, 어떤 요소의 존재 여부만 중요하고, 그 순서나 중복이 문제되지 않는 경우가 이에 해당합니다.


2. Map을 사용하는 경우 : 키와 값의 쌍으로 데이터를 관리해야 할 때 Map을 사용합니다. 이는 사전처럼 어떤 키(key)에 대한 값(value)을 빠르게 찾아야 할 때 유용합니다. 예를 들어, 사람의 이름을 키로, 그 사람의 나이를 값으로 저장하는 경우가 있겠죠.


3. 메서드에 익숙해지기 : Set과 Map을 더 효율적으로 사용하기 위해서는, 이들의 주요 메서드들에 익숙해져야 합니다. 예를 들어, Set의 add(), remove(), contains() 메서드와 Map의 put(), get(), containsKey() 메서드 등이 있습니다. 이러한 메서드들을 적절히 사용하여 데이터를 관리하세요.


4. 실습을 통한 학습 : 집합과 맵에 대해 배운 후에는, 간단한 문제부터 시작하여 점차 다양한 문제에 적용해 보는 것이 좋습니다. 처음에는 간단한 데이터 셋을 가지고 연습하면서 메서드들이 어떻게 작동하는지를 확인해 보세요.


5. 문제 해결 시도 : 실제 문제를 풀면서 집합과 맵을 언제 사용해야 하는지, 그리고 어떻게 사용해야 하는지를 파악해 보세요. 처음에는 어렵게 느껴질 수 있지만, 점차 직관적으로 어떤 자료구조를 사용해야 할지 감이 올 것입니다.
